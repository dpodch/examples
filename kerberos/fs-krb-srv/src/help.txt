API службы проверки и выдачи ключей керберос. Запросы принимаются только по 127.0.0.1.

1. Проверка ключа GET (/check)

Вход:
  Http заголовоки: 
    - Authorization - заголово содержащий ключ для проверки авторизации.

Выход:
  Http коды ответов:
    - 200 - пользователь по ключу авторизован
    - 400 - ошибка при формировании запроса
    - 401 - ошибка авторизации пользователя
  Http заголовки:
    - KRB_USER_NAME - имя пользователя
    - KRB_REALM - реалм пользователя
  Тело ответа:
    response:<ответ кербероса на запрос авторизации>
    username:<имя пользователя>
    realm:<реалм пользователя>

Пример запроса: 
curl --fail curl -H "Authorization: Negotiate <key>"  "http://127.0.0.1:8677/check"

2. Выдача ключа GET (/key)

Вход:
  Http заголовоки: 
    - AccessHost - хост для доступа к которому необходимо получить ключ

Выход:

  Http коды ответов:
    - 200 - ключ сформирован и выдан
    - 400 - ошибка при формировании запроса
    - 401 - пользователь не имеет доступа у указанному ресурсу

  Http заголовки:
    - Content-Length - длинна ключа

  Тело ответа:
    - ключ для доступа к AccessHost

Пример запроса:
curl -H "AccessHost: 1maoi04.dev.aorti.tech" "http://127.0.0.1:8677/key"
